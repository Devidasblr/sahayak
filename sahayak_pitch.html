Technology — Vapi

How we use Vapi

Vapi is the voice layer of Sahayak. It handles the entire call lifecycle — from receiving speech to speaking the final answer — so we can focus purely on the knowledge and logic.

01
Speech-to-Text in Hindi
We configure Vapi's transcriber with Deepgram Nova-2 set to language: "hi". This accurately transcribes rural Hindi dialects and code-switched speech (Hindi + English mix) that most STT engines fail at.
02
LLM Orchestration with a Custom Tool
Vapi runs GPT-4o as the brain. We give it a custom tool called search_knowledge that it must call before answering any factual question. This is how Vapi connects to our Qdrant database — via a webhook to our FastAPI server.
03
Indian Voice TTS
We use Azure's hi-IN-SwaraNeural voice through Vapi — a natural-sounding Indian female voice that users trust. The fallback is en-IN-NeerjaNeural for English queries, keeping the Indian accent consistent.
04
System Prompt Engineering
Sahayak's personality is defined in the Vapi system prompt: warm, simple language, always give a next-action step, mention helpline numbers, and never give dangerous medical advice without saying "go to hospital."

Technology — Qdrant

How we use Qdrant

Qdrant is the memory of Sahayak. Instead of relying on the LLM's training data (which may be outdated or hallucinated), we store curated, verified knowledge and retrieve it semantically at call time.

STEP 1

Ingest

Government scheme docs + healthcare guides are embedded using paraphrase-multilingual-MiniLM — a model that understands both Hindi and English text natively.

STEP 2

Store

Each document is stored in Qdrant as a 384-dim vector with rich metadata payload: title, eligibility, benefits, helpline, and Hindi description.

STEP 3

Search

At call time, the user's query is embedded and a cosine similarity search retrieves the top 3 most relevant chunks in under 50ms.

STEP 4

Respond

The retrieved context is injected into the LLM's prompt. The LLM answers only from this context — no hallucination, always grounded in real scheme data.

# Why multilingual embeddings matter # User says (Hindi): "मुझे मुफ्त इलाज चाहिए" # Query sent: "free health treatment scheme" ← translated by LLM # Qdrant finds: Ayushman Bharat (score: 0.89) ✓ model = SentenceTransformer("paraphrase-multilingual-MiniLM-L12-v2") # ↑ supports 50+ languages, Hindi included results = client.search( collection_name="sahayak_knowledge", query_vector=model.encode(query).tolist(), limit=3, # top 3 most relevant docs with_payload=True # returns full scheme metadata )

Why It Matters

Real societal impact

Sahayak isn't a chatbot. It's a bridge between India's most vulnerable citizens and the welfare systems built for them.

🗣️

Zero literacy required

Voice-first design means even a user who cannot read or type can access full government scheme guidance.

🌐

Multilingual by architecture

The multilingual embedding model understands queries in Hindi. Adding Kannada, Tamil, or Bengali requires only new data — no code changes.

🧠

Grounded, not hallucinated

Qdrant RAG ensures every answer comes from verified scheme data. The LLM cannot invent eligibility rules or wrong helpline numbers.

⚡

Extensible in minutes

Add a new government scheme by dropping a JSON file and running ingest.py. No retraining, no fine-tuning needed.
